# Cubic Solver

一个"跟着屏幕拧就能复原"的魔方 Web 应用。目标用户：不会魔方、或者曾经会但现在只能拼出底面的人——不用去搜教程、不用自己摸索，一步步跟着做就能还原。

## 运行

纯前端、零构建。用任意静态服务器起一个本地服务即可（ES module + importmap 从 CDN 加载 three.js）：

```bash
cd cubic-solver
python3 -m http.server 8173
# 打开 http://localhost:8173
```

> 直接双击 `index.html`（`file://`）打不开，浏览器会拦截 ES module，必须走 http。

## 现在能做什么（Phase 1 — demo）

- 经典配色的精致 3D 魔方（圆角块、贴纸、软阴影、环境光反射）。
- **拧一层**：直接在魔方上**拖动**任意一层（像真魔方），或用底部按钮 / 键盘 `U D L R F B`（`Shift` 逆时针）。
- **整体旋转**：按钮 `x y z` 或键盘 `X Y Z`（`Shift` 反向）。
- **自由观察 Free**：右上角开关（或空格），拖动 360° 观察，滚轮缩放。
- 打乱 / 复原。
- 每一步都是平滑缓动动画，不是生硬跳转。

## 架构

一切皆**标准记号**（`U/D/L/R/F/B` + 整体 `x/y/z`，带 `'` 和 `2`）。`RubiksCube`
把每个动作压进一个串行队列，动画引擎逐个消费——手动拧、拖动拧、以后的
solver 回放都共用这一套引擎。

- `src/cube.js` — 魔方建模 + 转动动画 + move 队列（27 个 cubie，旋转时临时挂到一个
  pivot 上转，转完再归位并对齐网格）。
- `src/main.js` — 场景 / 灯光 / 环境、free mode、按钮键盘、拖动拧交互。

## 路线图（Phase 2）

1. **编辑魔方**：把手上真实魔方各面颜色配置到 model 上（点色 + 合法性/可解性校验）。
2. **Solve**：初级层先法（LBL）——魔方转到合适起始面，在对应层高亮方向箭头，
   用户跟着做→确认→下一步（需要时旋转整体）。
3. 逐步引导直到复原。

## 已知 / 待验证

- 转动**方向**（顺 / 逆时针）用的是标准记号约定；拖动拧是几何自洽的应无问题，
  但记号按钮/键盘方向请开页面扫一眼确认，如反了只需翻转 `cube.js` 里对应 `FACE` 的 `baseSign`。
